# Google Analytics 4 (GA4) Configuration Guide

This guide details how to configure GA4 to properly track leads generated by the WhatsApp integration and Contact Forms.

## 1. Events Implemented

The application fires the following custom events via `gtag.js`:

### `lead_whatsapp_click`
Fired when a user clicks any WhatsApp contact button associated with a property.

**Parameters:**
- `event_category`: "Lead"
- `event_label`: "Venda" or "Locação" (based on property business type)
- `property_id`: UUID of the property
- `property_title`: Title of the property
- `property_type`: Type (apartment, house, etc.)
- `business_type`: 'sale' or 'rent'
- `value`: Property price
- `currency`: "BRL"

### `lead_form_submit`
Fired when a user successfully submits a contact form.

**Parameters:**
- `event_category`: "Lead"
- `event_label`: "Form Submission"
- `contact_method`: "form"
- `property_id`: UUID (if submitted from property page)
- `business_type`: 'sale' or 'rent' (if applicable)

## 2. Setting Up Conversions in GA4

To view these events as conversions (goals), follow these steps:

1.  **Log in to Google Analytics**.
2.  Go to **Admin** (Gear icon) -> **Property** -> **Events**.
3.  You will see a list of existing events. The new events (`lead_whatsapp_click` and `lead_form_submit`) will appear here **only after they have been triggered at least once**.
    *   *Tip: Use the application to fire these events yourself, then wait 24 hours.*
4.  Once they appear in the list, toggle the **"Mark as conversion"** switch to ON (blue).

### Manually Creating Events (If they don't appear yet)
1.  In **Admin** -> **Events**, click **Create event**.
2.  **Custom Event Name**: `generate_lead_whatsapp` (or any name you prefer for the conversion).
3.  **Matching Conditions**:
    *   `event_name` equals `lead_whatsapp_click`
4.  Click **Create**.
5.  Go to **Conversions** tab and click **New conversion event**.
6.  Enter `generate_lead_whatsapp` and save.

## 3. Viewing Real-Time Events

To verify that tracking is working immediately:

1.  Go to **Reports** -> **Realtime**.
2.  In the "Event count by Event name" card, you should see `lead_whatsapp_click` or `lead_form_submit` appear seconds after you perform the action on the site.
3.  Click on the event name to see the parameters (e.g., check if `business_type` is correct).

## 4. Filtering by Business Type (Sale vs Rent)

You can create specific audiences or segments to analyze Sale vs Rent leads separately.

**Example: Sale Leads Segment**
1.  Go to **Explore** -> **Free form**.
2.  In **Segments**, create a new **Event segment**.
3.  Condition: `event_name` = `lead_whatsapp_click` AND `business_type` = `sale`.
4.  Save as "WhatsApp Leads - Sale".

## 5. Validation Steps

1.  Open the website in a new incognito window (to avoid admin filtering if set up).
2.  Open Chrome Developer Tools -> Console.
3.  The application logs `[GA Dev] Event fired: ...` when running in development mode (localhost).
4.  Click a "Tenho Interesse" (WhatsApp) button on a property.
5.  Check the console for the log.
6.  Check GA4 Realtime view.
